## 2025-04-04 - N+1 Query in Google Sheets Synchronization
**Learning:** The Google Sheets API integration inside `lib/services/google_drive_service.dart` used an N+1 call pattern (`appendAttendanceRow` in a loop) for syncing history. This is extremely inefficient and slow, making it a classic backend performance bottleneck inside the frontend service logic. Replacing the loop with a single `sheetsApi.spreadsheets.values.append` drastically improves sync time. Additionally, when returning a nested typed list like `List<List<Object>>` in Dart, explicitly typing the inner literal as `<Object>[...]` is necessary to satisfy static analysis constraints.
**Action:** Always verify loops performing external API calls (especially Sheets or database inserts). Favor batch operations like `appendAttendanceRows` using `List<List<Object>>` to mitigate N+1 bottlenecks.
